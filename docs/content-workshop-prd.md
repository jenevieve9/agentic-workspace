# 内容工坊 · 产品 PRD（MVP → 迭代）

> 定位：AI 辅助内容生产的实验场。当前为 web 端产品原型，后续迭代为完整内容中台。
> 给研发的交付物：功能清单 + 优先级 + 技术需求 + 数据模型。

---

## 0. 总体架构与代码维护建议

- **现状**：已是 React + Vite SPA，按路由懒加载（`src/routes/index.jsx` + `src/pages/*`）。
- **数据层**：当前各模块用 Zustand + `persist` 存 localStorage；后期替换为 IndexedDB 或后端 API，store 接口不变即可平滑迁移。
- **模块边界**：减肥修身 / 日记 / 内容工坊 各自独立文件 + 独立 store，互不直接耦合，便于维护。
- **不拆多站**：数据需互通（思考积累→文章→发布），拆多站维护成本更高；保持统一数据层，前端按模块拆分即可。
- **后续扩展**：若功能膨胀，可引入微前端（qiankun）或按子应用部署，但 MVP 阶段不值得。

## 1. 模块一：IP 文章撰写

### 1.1 当前工作服务 IP 账号（启动路径）
自动化工作流：
```
选题筛选（灵感库 + 联动思考积累）
  → 提炼方向
  → 复刻什么内容
  → 自动化阐述
  → 人工优化
  → 自动化封面策划
  → 发布
  → 数据追踪
```

### 1.2 自建新账号（后续补充）
确定定位 / 人群 / 赛道 → 自动化生产工作流（待补充）。

### 1.3 完整 PRD（迭代目标）
| 功能 | 描述 | 优先级 |
| --- | --- | --- |
| 灵感库 | 爬取对标账号 + 手动收录 + 自动打标签 | P0 |
| 拆解引擎 | 输入对标内容 → 输出核心信息结构（观点/论据/钩子） | P0 |
| 定位配置 | 新账号：人群/赛道/风格 → AI 生成策略 | P1 |
| 工作流编排 | 选题→提炼→复刻→人工优化→封面→发布→数据追踪 | P0 |
| 多格式输出 | 同一想法 → 文章/口播脚本/图文/推文（参考 Dan Koe「一想法多形态」） | P1 |
| 数据看板 | 阅读/互动/转化追踪 | P2 |

### 1.4 技术需求
- 爬虫服务（Python + Scrapy）采集对标内容；
- AI 接口（DeepSeek / Gemini）做拆解与生成；
- 状态机管理内容生产流程；
- 数据库存储灵感库与历史内容。

## 2. 模块二：自动化排版 + 设计

### 2.1 商业化路径（分阶段）
| 阶段 | 目标 | 产出 |
| --- | --- | --- |
| 1 | 个人品牌视觉统一 | 视频号封面模板、公众号排版规范、个人简历设计 |

- Canva API 或 Figma API 集成；
- 模板系统（JSON 配置样式变量）；
- 批量导出（PDF / PNG）。

## 3. 模块三：自动化剪辑

### 3.1 工作流
```
素材 → 粗剪（video-use）→ 字幕生成（caption-clip）
  → 包装（模板）→ 切片（长切短）→ 发布 → 数据复盘
```

### 3.2 场景与工具
| 场景 | 工具 / 方案 | 商业化方向 |
| --- | --- | --- |
| IP 口播视频 | talking-head-editor + 口播文案 AI 生成 | 代运营服务 |
| AI 剧（宠物/幻想/苗疆/纪录片/口述历史） | 爬爆款画面 + AI 生成画面 + 配音 | 内容 IP 孵化 |
| 产品发布片 | product-launch-video 模板 | 企业服务 |
| 完整视频工作台 | claude-code-video-toolkit | 订阅制工具 |

### 3.3 技术需求
- 视频处理服务（FFmpeg 封装）；
- AI 生成脚本（DeepSeek）；
- 字幕生成与样式配置；
- 发布接口（抖音 / 小红书 Open API）。

---

## 4. 给研发的落地建议（MVP 切法）
1. 不一次全装，先跑通「IP 文章撰写」P0 链路（灵感库→拆解→生成→人工优化）；
2. 每个模块独立 store + 独立页面，先 localStorage 后接 API；
3. 工具链（video-use / caption-clip 等）先以「占位 + 接口预留」形式进入，验证流程再深做；
4. 数据看板最后做，先有内容生产再说追踪。
